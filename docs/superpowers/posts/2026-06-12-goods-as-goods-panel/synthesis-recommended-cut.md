### 1.2 The market

**The Exchange (OD-2).** One config-named corp is the seller/buyer-of-record for goods at EVERY station **including the haven**. In scenario_bazaar, Exchange == Port (== Yard, per the solvency arithmetic below). Four new credit legs, all pure transfers inside Σtreasury+Σwallets+Σescrow: package BUY (corp→Exchange at load-tick price, at `try_load`), package SELL (Exchange→corp at delivery-tick price, **saturating at the Exchange — goods still unload, contract still completes**), trader BUY (wallet→Exchange), trader SELL (Exchange→wallet, saturating). Goods legs: package = the existing load/unload transfers; trader = stock↔hold transfers, NO counter. Every settle clones the refuel `if unit_price < 1 { continue; }` guard (economy.rs:1023-1025) — L2-C3.

**Solvency honesty (L1-critique M1, attached to OD-2):** the loop does NOT close — worst Exchange drain ≈ 5.4e9 micros/100k ticks vs ≤0.3e9 refuel recapture; ransom wallets and Yard fees are true sinks. v1 ships option (a) as a sized battery with lossy recycling: merge Yard revenue into the Exchange, seed from a calibration run's measured drain window, and print the Exchange-drain as an anchored read. Consumption-minted money (the only self-sustaining variant) is the named trigger if the console shows universal late-game heat death.

**The trigger (all i64, saturating):** per corp/good/directed pair: `spread·qty − transport[route] − premium[corp] > 0` AND funding headroom `treasury ≥ wage + price[a]·qty` tracked through a per-scan `committed[c]` scratch (L1-M4). **transport[route] is a FACTORY-TIME integer table** from phase-independent ring-radius geometry (the tier-reward precedent, scenario.rs:466-467), folded as config — NOT a runtime ephemeris read. *Conflict resolved:* L2's static table beats L1's post-tick ephemeris distance — it kills the WB1 distance-oscillation confound (L1-critique M3a) and keeps the poster f64-free. Posted wage = `transport[route] + surplus × wage_share_milli / 1000` (OD-4). qty = smallest-first ladder. Premium floor derived per good against its ACTUAL cap (premium ∝ 1/cap — L1-m4).

**The poster (stage 1b2 slot):** one O(rows) in-flight scratch incl. source-side `outbound[a][g]` (never the REPOST O(rows²) rescan — G2#12); route-major deterministic iteration; **corp rotation `(route_index + scan_index) % n_corps` for first refusal** (L1-C2); one post per route per scan + MAX_POSTS_PER_SCAN backstop; projection damps both ends. `scan_interval > 0` gate precedes the modulo (L1-m2). Emits **ContractOffered only** — PackagePosted is DROPPED from spec §6's event list (it double-emits with the existing path, economy.rs:512-515; the `"post"` gossip-log row is runner-enriched from the contract row + current prices). *Conflict resolved:* L4-critique F3 beats L4's PackagePosted.

**Withdrawal (extends L1 ND-3 per L1-C1):** the scan FAILS (a) unclaimed Offered rows whose recheck no longer clears at current prices (exact integer formula in the plan: completing now loses money), and (b) **Accepted-but-never-loaded rows whose corp cannot fund the buy** — fail + escrow refund + release hauler (the settle_contract_failure legs minus the cargo leg). Emits unhashed `OfferWithdrawn`. Without (b), every corp bankruptcy permanently attrits the fleet (load no-ops forever; FuelEmpty is the only other exit, economy.rs:1376-1401).

**REPOST retirement:** config `demand_low = demand_high = 0` is the structural off (verified: burst needs `stock < demand_low`, top-up needs `projected < max(high,low)` — economy.rs:488-516); plus a hash-neutral early-return prelude commit (behavior-identical, proven by within-build digest) so the dead O(rows²) scan stops running over a growing board. Frontier's 10/20/16 untouched.

**Two-mode policy (D6):** `best_wage_net = ASSIGN score − transport[route]`; `best_trade_net = spread·qty_aff − transport[route]` — **the same table subtracted from BOTH sides** (L1-M2; the PDR-0007 seam as one shared function). Own-trade iff capitalized (`wallet ≥ buy + trade_reserve`) AND `best_trade_net > best_wage_net`; ties → wage. Day-0 wallets are 0 (verified world.rs:311) so WA3 opens at share 0 — that IS the story; no initial-wallet knob. Buy/sell verbs: `pending_trade_buy/sell` intent (pending_upgrade payload shape) in the 1c3x block, settle in 1dx, prev-frame dock predicate, refuel quantization, same-tick nav re-derivation; **pirates and !scripted craft skipped** (the refuel-policy precedent — without this rung-A pirates become own-traders and D7's split silently breaks, L2-critique #5). `TradeBought`/`TradeSold` land in the same commit that DELETES the dead `EventKind::Trade` (contract.rs:75-80, sole constructor :417). **ASSIGN gates package claims on an empty hold** (L3-M3: keeps the prey taxonomy exact in rung B and the no-double-rob invariant true).
