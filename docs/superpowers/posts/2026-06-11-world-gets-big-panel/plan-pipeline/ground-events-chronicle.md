2. `settle_contract_failure` takes no `tick`/`events` (economy.rs:1262-1268) and ZEROES `escrow_micros` mid-body (:1296): grow the signature, capture `escrow_refunded_micros` before the refund leg, and cause-gate the emit so the Robbed caller (pirate.rs:236) does not double-narrate.
3. Events are hash-neutral by design (contract.rs:97-98) — but anything you make the EMIT depend on must already be hashed state; replay compares `(tick, state_hash)` streams only (trophic_run.rs:452-475), so a divergent event stream with identical hashes would NOT be caught by replay-check.
